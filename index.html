第一章：Linux 結構

Linux，如你可能已經知道的，是一個用於個人電腦、伺服器甚至行動裝置的作業系統。然而，Linux 在資安領域中更是一個基石，以其穩定性、彈性以及開源特性而聞名。在本章中，我們將介紹 Linux 的結構、歷史、哲學、架構及檔案系統層次—這些都是任何資安專業人員的基礎知識。你可以將這個章節想像成學開新車的第一堂駕訓課，了解整台車的組成及運作原理。

什麼是 Linux？

Linux 是一種作業系統，就像 Windows、macOS、iOS 或 Android。作業系統（OS）是一種管理電腦硬體資源的軟體，使軟體應用程式與硬體之間可以順利溝通。與其他作業系統不同，Linux 有許多不同的版本，稱為「發行版」（Distro），這些版本針對不同需求和偏好進行調整。

歷史

Linux 的誕生與多個重要事件息息相關。1970 年，Ken Thompson 與 Dennis Ritchie 發布了 Unix 作業系統。1977 年，Berkeley Software Distribution（BSD）發行，但因為包含 AT&T 擁有的 Unix 程式碼，開發受到訴訟限制。1983 年，Richard Stallman 發起 GNU 計畫，目標是打造自由的 Unix 類作業系統，並創立 GNU 通用公共授權（GPL）。

Linux 本身於 1991 年由芬蘭學生 Linus Torvalds 作為個人專案開始，目標是創建一個自由的作業系統核心（Kernel）。經過多年發展，Linux 核心從最初少量 C 程式碼，演變成擁有超過 2300 萬行程式碼、廣泛使用且採用 GPL v2 授權的作業系統。

Linux 有超過 600 種發行版，包括 Ubuntu、Debian、Fedora、OpenSUSE、elementary、Manjaro、Gentoo、RedHat 和 Linux Mint。Linux 通常被認為比其他作業系統更安全、更穩定、效能更高，並廣泛運行於伺服器、桌面、嵌入式系統，甚至 Android 智慧型手機上。

哲學

Linux 的哲學核心是簡單性、模組化與開放性。它提倡構建小型、單一功能的程式，並可透過組合完成複雜任務，提高效率與彈性。主要原則如下：

第二章：Linux 發行版

Linux 發行版（或稱 Distro）是基於 Linux 核心的作業系統。它們用途多樣，從伺服器、嵌入式裝置到桌面電腦與行動裝置都能使用。Linux 發行版就像同一家公司下的不同分支或加盟店，每個分支針對特定市場或用戶需求進行調整。雖然它們共享相同的核心元件（components）、架構（architecture）與哲學（philosophy），但每個發行版都有其獨特的軟體包與設定，為不同需求提供客製化體驗——同時遵循 Linux 的統一品牌與價值觀。

每個 Linux 發行版都不同，擁有各自的功能、套件與工具。一些常見的發行版包括：

  • Ubuntu
  • Fedora
  • CentOS
  • Debian
  • Red Hat Enterprise Linux

許多使用者選擇 Linux 作為桌面系統，因為它免費、開源且高度可自訂。Ubuntu 與 Fedora 是桌面與新手 Linux 的熱門選擇。Linux 也被廣泛用於伺服器，因為它安全、穩定、可靠且定期更新。對於資安專家而言，我們偏好 Linux 的原因在於它是開源的，程式碼可供檢視與自訂。這樣我們可以針對特定需求最佳化與設定 Linux 發行版。

Linux 發行版的使用範圍極廣，包括網頁伺服器、行動裝置、嵌入式系統、雲端運算與桌面電腦。對資安專家而言，一些熱門的 Linux 發行版包括（但不限於）：

不同 Linux 發行版的主要差異在於預設套件、使用者介面與可用工具。Kali Linux 是資安專家最常用的發行版，內建大量安全工具與套件。Ubuntu 在桌面使用者間很普及，而 Debian 則在伺服器與嵌入式系統中廣受歡迎。Red Hat Enterprise Linux 與 CentOS 則多用於企業級計算環境。

Debian

Debian 是廣受好評的 Linux 發行版，以穩定性與可靠性著稱，用途包括桌面電腦、伺服器及嵌入式系統。它使用 Advanced Package Tool（apt）套件管理系統來處理軟體更新與安全修補程式。套件管理系統可自動下載並安裝安全更新，保持系統最新且安全，也可手動操作。

Debian 的學習曲線可能比其他發行版稍陡，但它被認為是最靈活、可自訂性最高的 Linux 發行版之一。設定與安裝可能較複雜，但它提供對系統的高度控制，對進階使用者非常有利。雖然初學者會感覺繁瑣，但一旦掌握基本指令與工具，操作效率反而更高。

Debian 的穩定性與可靠性是其核心優勢。它的長期支援（LTS）版本可提供長達五年的更新與安全修補，對 24/7 伺服器尤為重要。Debian 的開發社群在發現漏洞後會迅速釋出修補程式，並且對安全與隱私有高度承諾。總體而言，Debian 是一個多用途、可靠且安全的 Linux 發行版，對各種用途，包括資安工作，都非常適合。

第三章：Shell 介紹

學習如何使用 Linux Shell 非常重要，因為許多伺服器都是基於 Linux 的。Linux 伺服器通常比 Windows 伺服器更穩定且更少出錯，例如許多網站伺服器都基於 Linux。要有效地控制作業系統，必須理解並掌握 Linux 的核心部分——Shell。

Linux 終端機，也稱為 Shell 或命令列，提供使用者與核心之間的文字輸入/輸出 (I/O) 介面。雖然「Console」一詞也常被使用，但它指的是文字模式的螢幕，而不是視窗。在終端機視窗中，我們可以執行指令來控制系統。

可以把 Shell 想像成一個文字化的 GUI，我們透過輸入指令來執行操作，例如切換目錄、管理檔案、取得系統資訊等，而且功能遠超過一般 GUI。

終端機模擬器

終端機模擬器是模擬終端機功能的軟體，它允許在圖形使用者介面 (GUI) 中使用文字程式。此外，也存在在一個終端機內運行額外終端機的命令列介面 (CLI)。簡單來說，終端機是與 Shell 互動的介面。

可以想像你在一棟大辦公大樓中，Shell 是主要的伺服器機房，負責處理所有公司的資料與指令。終端機就像接待處，你透過接待處（終端機）向伺服器機房（Shell）發送指令或請求。

如果你在遠端工作，終端機模擬器就像電腦螢幕上的虛擬接待處（GUI），讓你不必親自到辦公室，也能與伺服器機房互動。它模擬實際接待處的功能，允許在圖形環境中使用文字程式與指令。

另外，在單一終端機內運行的多個 CLI 就像在螢幕上同時打開多個虛擬接待處，每個接待處都可以獨立傳送指令到伺服器機房，但仍共用同一主介面。本質上，終端機是與 Shell 核心操作溝通與控制的入口。

終端機模擬器與多工器（Multiplexer）是終端機的有用擴充功能，提供分割視窗、在多個目錄操作、建立不同工作區等多種方法與功能。例如，多工器 Tmux 的使用可以將終端機分成多個窗格，同時管理不同目錄：

Shell

Linux 最常用的 Shell 是 Bourne-Again Shell（BASH），它是 GNU 專案的一部分。透過 Shell，我們可以執行 GUI 可以完成的操作，並能以更快的方式與程式或流程互動。此外，許多流程可以透過撰寫小型或大型腳本自動化，大幅減少手動操作。

除了 Bash，還有其他 Shell，如 Tcsh/Csh、Ksh、Zsh、Fish Shell 等。

第四章：提示符說明

Bash 提示符（Prompt）非常容易理解。預設情況下，它會顯示使用者名稱（你是誰）、電腦名稱（hostname）以及目前所在的資料夾/目錄。提示符是一行文字，顯示在螢幕上，告訴使用者系統已準備好接收指令。提示符出現在新行，游標（閃爍的線或方框）會緊跟其後，等待使用者輸入指令。

提示符可以自訂，以提供使用者更多有用的資訊。其格式可能如下：

  @$

使用者的主目錄以波浪號 ~ 表示，這是登入時的預設資料夾：

  @[~]$

美元符號 $ 表示普通使用者。如果以 root 登入，符號會變成井號 #，例如：

  root@htb[/htb]#

例如，在目標系統上上傳並執行 shell 時，我們可能看不到使用者名稱、主機名稱與當前目錄。這通常是因為環境變數 PS1 未正確設定。在這種情況下，提示符可能顯示如下：

普通使用者 Shell 提示符：

  $

管理員（root）Shell 提示符：

  #

在 Linux 系統中，PS1 變數控制命令提示符在終端機中的顯示方式。它就像一個模板，每次系統準備好接收指令時，會依據這個模板顯示文字。透過自訂 PS1，可以讓提示符顯示使用者名稱、電腦名稱、目前資料夾，甚至加入顏色與特殊符號，讓命令列更具資訊性或美觀。

除了基本資訊，提示符還可以顯示 IP 位址、日期、時間，以及上一個命令的執行成功或失敗情況。這對於滲透測試特別有用，因為可以更有效地追蹤操作。例如，可以設定提示符顯示完整路徑而非僅名稱，甚至包含目標 IP 位址。使用 script 或查看使用者主目錄下的 .bash_history，可以記錄所有使用過的指令，並依日期時間整理，方便文件化與分析。

  • \u：當前使用者名稱

  • \w：當前工作目錄完整路徑

除了提示符，使用者還可以自訂終端機環境，例如色彩主題、字型等，使工作環境更美觀、易用。

第五章：尋求幫助

在建立了 Linux 結構、各種發行版以及 Shell 的基礎知識之後，我們現在可以將這些知識付諸實踐。是時候直接在終端機中使用指令，並學習在遇到不熟悉的指令時，如何尋求幫助。

在使用 Linux 時，我們總會碰到一些工具，其可選參數不熟悉，或者完全沒見過的新工具。因此，了解如何自助尋求幫助、熟悉這些工具是非常重要的。最常用的兩種方式是：man 手冊頁和 help 功能。先熟悉工具的基本用法，能讓我們更有效地操作，並可能發現一些原本以為不可行的技巧。在 man 手冊中，我們可以找到詳細說明和範例。

第一個範例指令

Bryan4698@htb[/htb]$ ls

cacert.der  Documents  Music     Public     Videos
Desktop     Downloads  Pictures  Templates

在 Linux 和 Unix 系統中，ls 指令用來列出當前資料夾或指定資料夾中的檔案與目錄，方便使用者管理。像大多數 Linux 指令一樣，ls 提供了額外選項，幫助使用者過濾或格式化輸出，顯示所需資訊。

使用 man 指令尋求幫助

要了解某個工具或指令提供哪些選項，可以使用 man 指令查看手冊頁：

Bryan4698@htb[/htb]$ man

例如查看 ls 的手冊：

Bryan4698@htb[/htb]$ man ls

LS(1)                            User Commands                           LS(1)

NAME
       ls - 列出目錄內容

SYNOPSIS
       ls [OPTION]... [FILE]...

DESCRIPTION
       列出 FILE 的資訊（預設為當前目錄）。
       若未指定 -cftuvSUX 或 --sort，則按字母排序。

       長選項的必填參數同樣適用於短選項。

       -a, --all
              不忽略以 . 開頭的檔案

       -A, --almost-all
              不列出隱含的 . 和 ..

       --author
       ...

使用 --help 或 -h 簡短幫助

我們也可以快速查看指令可選參數，而不需完整瀏覽手冊：

Bryan4698@htb[/htb]$  --help

例如：

Bryan4698@htb[/htb]$ ls --help

第十一章：檔案描述符與重定向

在 Unix/Linux 系統中，檔案描述符（File Descriptor, FD）是核心維護的一個引用，用於管理 I/O（輸入/輸出）操作。它可被視為對開啟檔案、套接字或其他 I/O 資源的唯一識別號。在 Windows 系統中，相對應的是「檔案句柄（File Handle）」。

想像成在衣帽間掛衣服時領到的票號：票號（FD）代表你對衣服（檔案/資源）的連結，每次取衣服（I/O 操作）時交給管理員（作業系統），即可快速找到目標。沒有票號，就無法有效存取對應資源。

Linux 預設前三個檔案描述符

  • STDIN – 0：標準輸入
  • STDOUT – 1：標準輸出
  • STDERR – 2：標準錯誤輸出

STDIN 與 STDOUT 範例

Bryan4698@htb[/htb]$ cat
SOME INPUT
# 輸入後按 [ENTER]，輸出即為 STDOUT

STDOUT 與 STDERR 範例

Bryan4698@htb[/htb]$ find /etc/ -name shadow
# STDOUT 會列出 /etc/shadow
# STDERR 顯示 Permission denied

將 STDERR 重定向至 /dev/null

Bryan4698@htb[/htb]$ find /etc/ -name shadow 2>/dev/null
# 標準錯誤訊息將被丟棄

將 STDOUT 重定向到檔案

Bryan4698@htb[/htb]$ find /etc/ -name shadow 2>/dev/null > results.txt
Bryan4698@htb[/htb]$ cat results.txt
# 只包含 STDOUT

將 STDOUT 與 STDERR 分開重定向到不同檔案

Bryan4698@htb[/htb]$ find /etc/ -name shadow 2> stderr.txt 1> stdout.txt
# STDOUT → stdout.txt
# STDERR → stderr.txt

STDIN 重定向範例

Bryan4698@htb[/htb]$ cat < stdout.txt
# 透過 STDIN 輸入檔案內容

STDOUT 追加到檔案

Bryan4698@htb[/htb]$ find /etc/ -name passwd >> stdout.txt 2>/dev/null
# 使用 >> 追加 STDOUT 到已存在檔案

STDIN 流重定向（Here Document）

Bryan4698@htb[/htb]$ cat << EOF > stream.txt
Hack The Box
EOF
Bryan4698@htb[/htb]$ cat stream.txt
# stream.txt 內容為 "Hack The Box"

使用管線（Pipes）處理 STDOUT

Bryan4698@htb[/htb]$ find /etc/ -name *.conf 2>/dev/null | grep systemd
# 將 STDOUT 送到 grep 過濾包含 "systemd" 的行

Bryan4698@htb[/htb]$ find /etc/ -name *.conf 2>/dev/null | grep systemd | wc -l
# 計算符合條件的行數

透過理解檔案描述符、重定向及管線，我們可以更精確地控制資料的流向，將輸入、輸出與錯誤訊息分別導向檔案或其他程式，使操作更有效率，也更方便處理大量資料。

第十三章：正規表達式 (Regular Expressions)

正規表達式（RegEx）就像為文字或檔案設計精準藍圖的藝術。它允許我們以極高的精確度搜尋、替換或操作資料。可以把 RegEx 想成一個高度可客製化的過濾器，用來篩選字串，找出你需要的內容—無論是分析資料、驗證輸入，還是進行進階搜尋。

正規表達式的核心是一連串的字元與符號，組成搜尋模式。這些模式通常包含特殊符號（metacharacters），它們定義搜尋的結構，而不是字面文字。例如，metacharacters 可以讓你指定搜尋數字、字母或符合特定模式的任意字元。

RegEx 廣泛應用於許多程式語言與工具中，如 grep、sed，是我們工具箱中非常強大的工具。

分組 (Grouping)

RegEx 提供了將搜尋模式分組的功能，主要使用三種括號：

運算符 說明
(a) 小括號用於將 RegEx 部分分組，括號內的模式會一起處理。
[a-z] 中括號用於定義字元類別，列出要搜尋的字元集合。
{1,10} 大括號用於定義量詞，指定前一模式重複次數或範圍。
| OR 運算符，當任一表達式符合時會顯示結果。
.* 類似 AND 運算符，只有當兩個表達式依指定順序同時匹配時才顯示結果。

OR 運算符範例

cry0l1t3@htb:~$ grep -E "(my|false)" /etc/passwd

lxd:x:105:65534::/var/lib/lxd/:/bin/false
pollinate:x:109:1::/var/cache/pollinate:/bin/false
mysql:x:116:120:MySQL Server,,:/nonexistent:/bin/false
# 匹配包含 "my" 或 "false" 的行

AND 運算符範例

cry0l1t3@htb:~$ grep -E "(my.*false)" /etc/passwd

mysql:x:116:120:MySQL Server,,:/nonexistent:/bin/false
# 只有同時包含 "my" 與 "false" 的行才匹配

簡化 AND 範例（管線）

cry0l1t3@htb:~$ grep -E "my" /etc/passwd | grep -E "false"

mysql:x:116:120:MySQL Server,,:/nonexistent:/bin/false

以下練習將使用 /etc/ssh/sshd_config 檔案，幫助你熟悉 RegEx 的實際應用：

  1. 顯示所有不包含 # 的行。
  2. 搜尋所有包含以 Permit 開頭單字的行。
  3. 搜尋所有包含以 Authentication 結尾單字的行。

建議多練習 RegEx，嘗試組合不同的運算符、分組與量詞，並搭配 grep -E 或其他工具，將會提升文字與檔案資料處理能力。

第十四章：權限管理 (Permission Management)

在 Linux 中，權限就像控制檔案與目錄存取的鑰匙。這些權限會分配給使用者與群組，每個使用者可以屬於多個群組，群組成員會獲得額外的存取權限。每個檔案與目錄都有一個擁有者（user）與對應的群組，並依此設定可讀、可寫或可執行的權限。

當使用者要進入目錄時，需要執行權限（x）才能「遍歷」目錄。即便內容可見，沒有 execute 權限也無法進入，會顯示「Permission Denied」。

範例：檢視權限

cry0l1t3@htb[/htb]$ ls -l
drw-rw-r-- 3 cry0l1t3 cry0l1t3 4096 Jan 12 12:30 scripts

cry0l1t3@htb[/htb]$ ls -al mydirectory/
ls: cannot access 'mydirectory/script.sh': Permission denied
-????????? ? ? ? ? ? script.sh
d????????? ? ? ? ? ? subdirectory

執行權限允許進入目錄，但不代表可修改或執行其中檔案。修改檔案需要檔案本身的 write 或 execute 權限。

檔案權限結構

cry0l1t3@htb[/htb]$ ls -l /etc/passwd
-rwx rw- r-- 1 root root 1641 May  4 23:42 /etc/passwd
# 權限對應：
# rwx | rw- | r--
# owner | group | others
# r = 讀, w = 寫, x = 執行
# - = 無權限, d = 目錄, l = 連結

修改權限 (chmod)

cry0l1t3@htb[/htb]$ ls -l shell
-rwxr-x--x 1 cry0l1t3 htbteam 0 May  4 22:12 shell

cry0l1t3@htb[/htb]$ chmod a+r shell && ls -l shell
-rwxr-xr-x 1 cry0l1t3 htbteam 0 May  4 22:12 shell

cry0l1t3@htb[/htb]$ chmod 754 shell && ls -l shell
-rwxr-xr-- 1 cry0l1t3 htbteam 0 May  4 22:12 shell

權限對應表

Binary Notation:          4 2 1  | 4 2 1 | 4 2 1
Binary Representation:    1 1 1  | 1 0 1 | 1 0 0
Octal Value:                  7   |   5   |   4
Permission Representation: r w x | r - x | r - -

更改擁有者 (chown)

cry0l1t3@htb[/htb]$ chown : 
cry0l1t3@htb[/htb]$ chown root:root shell && ls -l shell
-rwxr-xr-- 1 root root 0 May  4 22:12 shell

SUID & SGID

特殊權限 SUID/SGID 允許以檔案擁有者或群組的權限執行程式，類似臨時通行證。設定後會在 x 權限的位置顯示 s。

注意安全風險：若將 SUID 隨意賦予可開啟 shell 的程式，可能導致非授權使用者取得 root 權限。

Sticky Bit

Sticky bit 類似共享目錄的鎖，設定在目錄上時，只有檔案擁有者、目錄擁有者或 root 可以刪除或重新命名檔案。其他人可以讀取但無法修改。

cry0l1t3@htb[/htb]$ ls -l
drw-rw-r-t 3 cry0l1t3 cry0l1t3 4096 Jan 12 12:30 scripts
drw-rw-r-T 3 cry0l1t3 cry0l1t3 4096 Jan 12 12:32 reports
# 小寫 t = sticky bit + execute 權限
# 大寫 T = sticky bit，但其他使用者無 execute 權限

Sticky bit 特別適用於多使用者共享的目錄，防止未授權修改檔案。

第十五章：使用者管理 (User Management)

有效的使用者管理是 Linux 系統管理的重要基礎。管理員常需要建立新帳號或將現有使用者加入特定群組，以維護適當的存取控制。此外，執行命令時切換使用者身份也是常見需求，尤其是不同權限的任務。例如，某些群組可能專屬於檔案或目錄的存取權限，確保系統安全與完整性。

假設有新員工 Alex 加入公司，並獲得 Linux 工作站。系統管理員需建立 Alex 的使用者帳號，並將其加入有權存取專案檔案或開發工具的群組。若某些任務需要以不同使用者或更高權限執行，也需妥善處理。

以一般使用者執行

以 root 執行

在目標系統上，大部分操作皆可重置，因此可放心探索，直到對 Linux 系統操作感到熟悉與自信。

第十七章：服務與進程管理 (Service and Process Management)

服務（Service，又稱 daemon）是 Linux 系統中在背景運行的核心組件，無需直接使用者互動，但提供重要功能。服務通常可分為：

  • 系統服務：系統啟動時必須運行的內部服務，如硬體初始化，確保系統正常運作。
  • 使用者安裝的服務：由使用者安裝的服務，通常是伺服器應用程式或其他背景功能，提升系統功能性。

常見目標：

  • 啟動 / 重新啟動服務或進程
  • 停止服務或進程
  • 查看服務或進程狀態
  • 設定開機啟用 / 禁用
  • 查找服務或進程

第十八章：任務排程 (Task Scheduling)

Linux 系統的任務排程允許使用者或系統管理員自動化執行任務，於指定時間或固定間隔執行，省去手動操作。常見應用包括軟體自動更新、腳本執行、資料庫維護及備份。理解任務排程對資安專家和滲透測試員非常重要，可用於合法管理或偵測潛在的惡意活動。

Systemd 定時器

Systemd 可用於設定定時器與服務，自動執行腳本或進程：

建立 Timer

Bryan4698@htb[/htb]$ sudo mkdir /etc/systemd/system/mytimer.timer.d
Bryan4698@htb[/htb]$ sudo vim /etc/systemd/system/mytimer.timer
  
[Unit]
Description=My Timer

[Timer]
OnBootSec=3min
OnUnitActiveSec=1hour

[Install]
WantedBy=timers.target

使用 OnBootSec 可設定開機後延遲執行一次，使用 OnUnitActiveSec 可設定定期執行。

建立 Service

Bryan4698@htb[/htb]$ sudo vim /etc/systemd/system/mytimer.service
  
[Unit]
Description=My Service

[Service]
ExecStart=/full/path/to/my/script.sh

[Install]
WantedBy=multi-user.target

重新載入 Systemd 與啟用定時器

Bryan4698@htb[/htb]$ sudo systemctl daemon-reload
Bryan4698@htb[/htb]$ sudo systemctl start mytimer.timer
Bryan4698@htb[/htb]$ sudo systemctl enable mytimer.timer

此時 mytimer.service 將依照定時器設定自動執行。

Cron 排程

Cron 也可自動化任務。使用者需編輯 crontab，指定執行時間及腳本：

時間欄位說明
Minutes (0-59)任務執行的分鐘
Hours (0-23)任務執行的時
Days of month (1-31)任務執行的日
Months (1-12)任務執行的月
Days of the week (0-7)任務執行的星期
 
# 系統更新
0 */6 * * * /path/to/update_software.sh

# 每月第一天執行腳本
0 0 1 * * /path/to/scripts/run_scripts.sh

# 每週日清理資料庫
0 0 * * 0 /path/to/scripts/clean_database.sh

# 每週日備份
0 0 * * 7 /path/to/scripts/backup.sh

Cron 可搭配郵件通知或日誌監控任務執行結果。

Systemd vs Cron

Systemd 與 Cron 都可排程任務，但設定方式不同：

  • Systemd：需建立定時器 (.timer) 與服務 (.service) 腳本
  • Cron：需建立 crontab，交由 cron daemon 執行

第十九章：網路服務 (Network Services)

第二十一章：備份與還原 (Backup and Restore)

Linux 系統提供多種強大的備份與還原工具，如 Rsync、Duplicity、Deja Dup，用於保護資料不受遺失或破壞，並確保資料可隨時取回。

Rsync

Rsync 是一個開源工具，可快速備份資料，僅傳輸變更部分，特別適合網路傳輸和增量備份。

第二十二章：檔案系統管理 (File System Management)

管理 Linux 檔案系統是一項重要工作，涉及組織、存儲與維護資料。Linux 支援多種檔案系統，如 ext2、ext3、ext4、XFS、Btrfs、NTFS 等，每種系統具有不同特性，適合不同用途。

常見檔案系統介紹

  • ext2：較舊，無 journaling，適合低負載設備（如 USB）。
  • ext3 / ext4：支援 journaling，ext4 為現代 Linux 預設，平衡性能與可靠性。
  • Btrfs：支援 snapshot、資料完整性檢查，適合複雜存儲架構。
  • XFS：適合大檔案與高 I/O 環境。
  • NTFS：Windows 相容，適用於雙系統或外接磁碟。

Inode 與檔案類型

Linux 使用 inode 結構儲存檔案元資料（權限、擁有者、大小、時間戳等），inode 表則管理所有檔案與目錄。類比圖書館，inode 如索引卡，inode 表如整個目錄系統。

檔案類型：

  • Regular files：常見文字或二進位檔案
  • Directories：檔案容器，組織檔案
  • Symbolic links：快捷方式，指向其他檔案或目錄

第二十三章：容器化 (Containerization)

容器化是將應用程式打包並在隔離環境中運行的過程，這些容器提供輕量且一致的環境，確保應用程式在任何系統上都能以相同行為運行。常見技術包括 Docker、Docker Compose 和 LXC。

容器 vs 虛擬機

  • 共享主機核心，較輕量、效率高
  • 便於快速部署、管理和擴展
  • 隔離應用程式，提供安全屏障

Docker

Docker 是一個開源平台，用於將應用打包成容器。它使用分層檔案系統與資源隔離，提供靈活與可攜性。容器可以透過 Dockerfile 建立。

echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update -y
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin -y
sudo usermod -aG docker htb-student
docker run hello-world

第二十五章：Remote Desktop Protocols in Linux

Remote desktop protocols 在 Windows、Linux、macOS 中提供圖形化遠端存取系統，方便管理、維護與故障排除。常用的協議包括：

  • RDP (Remote Desktop Protocol)：主要用於 Windows，允許遠端桌面操作。
  • VNC (Virtual Network Computing)：跨平台協議，在 Linux 中常用，提供遠端桌面操作。

RDP 就像是 Windows 專用的鑰匙，VNC 則是通用鑰匙，但多用於 Linux 系統。

XServer

XServer 是 X Window System (X11) 的使用端部分，負責圖形介面與系統溝通。X11 支援 TCP/IP 傳輸，通常使用 TCP 6000-6009 端口，提供網路透明性。

若本地與遠端都是 Unix/Linux 系統，可直接使用 X11 避免 VNC/RDP。缺點是未加密，可透過 SSH 隧道加密。

# 啟用 X11 Forwarding
cat /etc/ssh/sshd_config | grep X11Forwarding
X11Forwarding yes

# 透過 SSH 執行遠端應用
ssh -X user@remote /usr/bin/firefox

X11 安全性

X11 預設未加密，TCP 6000-6010 端口容易被同網段攻擊者截取視窗資料，例如使用 xwd 或 xgrabsc 截圖，建議透過 SSH 隧道保護。

XDMCP

X Display Manager Control Protocol (XDMCP) 使用 UDP 177 端口，管理遠端 X Window session。可將整個 GUI 重定向，但不安全，容易遭 MITM 攻擊。

VNC

VNC 基於 RFB 協議，提供遠端桌面控制，支持跨平台，需身份驗證與加密。常用 TCP 端口 5900 + 顯示號（:0=5900, :1=5901 ...）。

TigerVNC 安裝與設定

# 安裝 XFCE 與 VNC Server
sudo apt install xfce4 xfce4-goodies tigervnc-standalone-server -y

# 設定 VNC 密碼
vncpasswd

# 建立必要設定檔
touch ~/.vnc/xstartup ~/.vnc/config

# xstartup 設定
cat <<EOT >> ~/.vnc/xstartup
#!/bin/bash
unset SESSION_MANAGER
unset DBUS_SESSION_BUS_ADDRESS
/usr/bin/startxfce4
[ -x /etc/vnc/xstartup ] && exec /etc/vnc/xstartup
[ -r $HOME/.Xresources ] && xrdb $HOME/.Xresources
x-window-manager &
EOT

# config 設定
cat <<EOT >> ~/.vnc/config
geometry=1920x1080
dpi=96
EOT

# 設定執行權限
chmod +x ~/.vnc/xstartup

# 啟動 VNC server
vncserver

# 列出 VNC sessions
vncserver -list

SSH 隧道加密連線

# 透過 VNC client 連線

第二十六章：Linux Security

所有電腦系統都有被入侵的風險。Linux 系統相對 Windows 系統病毒少、攻擊面小，但仍需落實安全措施。

基本安全措施

  • 保持作業系統與已安裝套件更新： 
    apt update && apt dist-upgrade
  • 設定防火牆與 iptables 限制進出流量
  • SSH 安全設定：
    • 禁止 root 登入
    • 避免使用密碼登入，建議使用金鑰驗證
  • 使用最小權限原則管理用戶與 sudo 權限
  • Fail2ban：限制連續登入失敗的 IP
  • 定期審計系統，檢查：
    • 過期內核
    • 用戶權限問題
    • 世界可寫檔案
    • cron 任務或服務錯誤配置

進階安全措施

  • SELinux / AppArmor：透過標籤與策略控制訪問權限
  • 常用安全工具：
    • Snort
    • chkrootkit
    • rkhunter
    • Lynis
  • 其他安全設定：
    • 移除或停用不必要的服務與軟體
    • 移除未加密認證的服務
    • 確保 NTP 啟用，Syslog 運行
    • 每位用戶擁有自己的帳號
    • 強制使用強密碼
    • 密碼老化與防止舊密碼重複使用
    • 登入失敗鎖定帳號
    • 停用不必要的 SUID/SGID 程式

TCP Wrappers

TCP Wrappers 允許管理員根據 IP 或主機名控制服務存取，設定檔如下：

/etc/hosts.allow
# 允許 SSH 從內網存取
sshd : 10.129.14.0/24

# 允許 FTP 從特定主機存取
ftpd : 10.129.14.10

# 允許 Telnet 從 inlanefreight.local 網域存取
telnetd : .inlanefreight.local

/etc/hosts.deny
# 拒絕來自 inlanefreight.com 網域的所有服務
ALL : .inlanefreight.com

# 拒絕 SSH 從特定主機
sshd : 10.129.22.22

# 拒絕 FTP 從 10.129.22.0/24 網段
ftpd : 10.129.22.0/24

注意：規則順序重要，第一個匹配的規則將被套用。TCP Wrappers 不替代防火牆，只控制服務存取而非端口。

第二十七章：Firewall Setup

防火牆主要用於控制與監控不同網路區段之間的流量，如內部網路與外部網路。它可以依規則、協議、端口等過濾進出流量，以防止未授權存取和安全威脅。

Linux 防火牆概述

iptables 是 Linux 早期防火牆工具，取代了 ipchains 與 ipfwadm。它透過 Netfilter 核心框架操作封包，提供靈活的防火牆規則設定。

iptables 主要組件

組件描述
Tables分類與組織防火牆規則
Chains規則鏈，用於處理特定類型的網路流量
Rules定義篩選封包的條件與動作
Matches匹配特定封包特徵，如 IP、端口、協議
Targets指定符合規則的封包處理方式，如 ACCEPT、DROP 等

Tables

表格名稱描述內建 Chains

額外的 raw 表格提供特殊封包處理選項，內建 PREROUTING 與 OUTPUT chains。

Chains

Chains 將規則組織起來，用於處理網路流量。分為內建與使用者定義：

  • filter 表格：INPUT, OUTPUT, FORWARD
  • nat 表格：PREROUTING, POSTROUTING
  • mangle 表格：PREROUTING, OUTPUT, INPUT, FORWARD, POSTROUTING
  • 使用者定義鏈：可依來源 IP、目的端口或協議分組規則

Rules & Targets

iptables 規則由 matches 與 target 組成，例如允許 SSH 流量：

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

第二十八章：System Logs

Linux 系統日誌包含系統與活動資訊，對於監控與排錯非常重要。分析日誌可協助識別安全弱點與異常活動，例如未授權登入、攻擊嘗試、明文憑證或異常檔案存取。

日誌用途

  • 監控系統行為、網路活動與使用者操作
  • 檢查安全測試效果與入侵事件
  • 透過適當設定、定期分析，提升系統安全

Linux 系統日誌類型

  • Kernel Logs (/var/log/kern.log)
  • System Logs (/var/log/syslog)
  • Authentication Logs (/var/log/auth.log)
  • Application Logs (如 /var/log/apache2/error.log)
  • Security Logs (如 /var/log/fail2ban.log, /var/log/ufw.log)

Kernel Logs

包含核心事件、硬體驅動與系統呼叫。可協助發現過期或有漏洞的驅動，以及系統異常。

第二十九章：Solaris

Solaris 是 Sun Microsystems（後被 Oracle 收購）開發的 Unix 系統，以穩定、可擴展、高階硬體支援聞名，常用於企業環境的關鍵應用，如資料庫、雲端運算與虛擬化。

Solaris vs Linux 發行版

主要差異：

  • Solaris 為專有系統，源碼不公開；Linux 多為開源。

Linux 提供許多快捷鍵，可以加快操作速度、減少輸入，甚至減少使用滑鼠的需求。

  • [TAB] - 自動補全。根據已輸入內容提供建議，如目錄名稱、命令或選項。
